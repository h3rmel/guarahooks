# Project Context - GuaraHooks

## Overview

**GuaraHooks** is a custom React hooks library built with Next.js, featuring comprehensive documentation, examples, and a CLI tool. This project uses a structured workflow to facilitate collaboration between human and AI in executing development tasks efficiently.

The library provides a collection of production-ready React hooks with TypeScript support, SSR compatibility, and comprehensive documentation.

## Project Architecture

### 🏗️ Technology Stack

- **Primary Language**: TypeScript
- **Framework**: Next.js 14+ (App Router)
- **Package Manager**: pnpm (mandatory)
- **Styling**: Tailwind CSS + Shadcn/ui
- **Documentation**: MDX with custom components
- **Monorepo Structure**: CLI package included

### 📁 Main Structure

```tree
guarahooks/
├── app/                    # Next.js App Router
│   ├── (docs)/            # Documentation routes  
│   └── showcase/          # Hook demonstrations
├── components/            # React components
│   ├── ui/               # Shadcn/ui primitives
│   ├── layout/           # Layout components
│   └── sections/         # Page sections
├── registry/             # Hook system core
│   ├── hooks/            # Hook implementations
│   ├── example/          # Usage examples
│   └── registry-*.ts    # Registration system
├── content/docs/         # MDX documentation
├── packages/cli/         # GuaraHooks CLI
├── lib/                  # Utilities
├── config/               # Project configuration
└── .cursor/              # Workflow system
    ├── stories/          # Task specifications
    ├── changelog/        # Implementation records
    └── rules/            # Project standards
```

## Workflow System

### 🔄 Development Process

1. **📝 Planning**: Human creates technical specification in `stories/` folder
2. **🤖 Execution**: AI implements following project rules and patterns
3. **📊 Documentation**: AI generates implementation record in `changelog/` folder
4. **✅ Validation**: Review against acceptance criteria

### 📋 Naming Standards

#### Stories (Technical Specifications)

- **Format**: `YYYY-MM-DD-task-description.md`
- **Example**: `2024-01-15-implement-use-local-storage-hook.md`
- **Content**: Technical requirements, file locations, code patterns

#### Changelog (Implementation Records)

- **Format**: `YYYY-MM-DD-task-description-CHANGELOG.md`
- **Example**: `2024-01-15-implement-use-local-storage-hook-CHANGELOG.md`
- **Content**: Auto-generated by AI documenting implementation details

### 🎯 Execution Principles

1. **Quality**: Follow established TypeScript and React standards
2. **Consistency**: Adhere to project patterns and conventions
3. **Documentation**: Include comprehensive documentation and examples
4. **Testing**: Implement appropriate tests and validation
5. **Performance**: Follow performance guidelines for React hooks

### 🔧 Core Development Commands

```bash

pnpm install                    # Install all dependencies

# Development
pnpm dev                       # Start development server
pnpm build                     # Build for production

# Quality

pnpm test                     # Run tests
```

### 🎨 Hook Development Flow

1. **Plan**: Define hook purpose, API, and implementation strategy
2. **Implement**: Create hook in `registry/hooks/use-{name}.ts`
3. **Example**: Create demo in `registry/example/use-{name}-demo.tsx`
4. **Document**: Create MDX documentation in `content/docs/hooks/`
5. **Register**: Add to registry system and navigation
6. **Test**: Verify functionality and build process

### 📊 Hook Categories

- **state-management**: State management hooks
- **ui-and-dom**: UI and DOM manipulation
- **utilities**: General utility hooks
- **lifecycle**: Component lifecycle hooks
- **data-fetching**: Network and data fetching
